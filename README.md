# Quiz SMSI: EBIOS RM & ISO 27001

Site web de révision dédié au cours de **SMSI**, **EBIOS RM** et **ISO/IEC 27001**, réalisé dans le cadre d’un Master 2 en cybersécurité.

L’objectif du projet est de permettre une révision rapide, claire et efficace avant un quiz papier, avec un format utilisable aussi bien sur ordinateur que sur téléphone.

## Objectif du projet

Ce site a été conçu pour aider à réviser les notions essentielles liées à :

* la méthode **EBIOS Risk Manager** ;
* la norme **ISO/IEC 27001** ;
* le fonctionnement d’un **SMSI** ;
* la gestion des risques ;
* les mesures de sécurité ;
* la gouvernance de la sécurité de l’information.

Le site propose un système de quiz avec plus de 100 questions, classées par thème et par niveau de difficulté, ainsi que des pages de révision synthétiques.

## Fonctionnalités

* Plus de 100 questions de révision
* Questions classées par niveau de difficulté :

  * Facile
  * Moyen
  * Difficile
  * Expert
* Questions organisées par thème :

  * EBIOS RM
  * ISO 27001
  * SMSI
  * Gestion des risques
  * Gouvernance
* Correction immédiate après chaque réponse
* Explication pédagogique pour comprendre la bonne réponse
* Page récapitulative du cours
* Page mémo avec les notions clés à retenir
* Interface responsive adaptée au mobile
* Navigation simple pour réviser rapidement
* Site utilisable en local sans serveur
* Compatible avec GitHub Pages

## Technologies utilisées

Le projet est réalisé avec des technologies simples et accessibles :

* HTML5
* CSS3
* JavaScript Vanilla

Aucun framework n’est nécessaire.

## Structure du projet

```text
.
├── index.html
└── README.md
```

Le fichier `index.html` contient directement :

* la structure HTML ;
* le style CSS ;
* le code JavaScript ;
* la banque de questions.

Cela permet d’avoir un site autonome, facile à lancer et à déposer sur GitHub.

## Lancer le projet en local

Pour utiliser le site en local :

1. Télécharger ou cloner le projet.
2. Ouvrir le fichier `index.html` dans un navigateur web.
3. Commencer les révisions.

Aucune installation particulière n’est nécessaire.

## Mise en ligne avec GitHub Pages

Pour publier le site avec GitHub Pages :

1. Créer un dépôt GitHub.
2. Ajouter le fichier `index.html` à la racine du dépôt.
3. Aller dans les paramètres du dépôt.
4. Ouvrir la section **Pages**.
5. Sélectionner la branche principale, généralement `main`.
6. Choisir le dossier racine `/root`.
7. Enregistrer.

GitHub génère ensuite automatiquement une URL publique pour accéder au site.

## Utilisation recommandée

Pour réviser efficacement :

1. Lire d’abord la page récapitulative du cours.
2. Consulter la page mémo pour retenir les définitions importantes.
3. Commencer les questions faciles.
4. Passer progressivement aux niveaux moyen, difficile puis expert.
5. Relire les explications des réponses fausses.
6. Refaire les questions jusqu’à obtenir un score stable.

## Objectif pédagogique

Ce projet n’a pas pour but de remplacer le cours, mais de fournir un support de révision rapide et interactif.

Il permet de vérifier la compréhension des notions principales, de mémoriser les définitions importantes et de s’entraîner dans un format proche d’un QCM.
